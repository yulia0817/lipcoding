# 개선 계획: 친구와 함께 타는 모닥불 (구 플랜-팀모드 개선판)

> 기존 `플랜-팀모드.md` 를 **개인 생산성 + 로그인 기반**으로 개선한 버전.
> 핵심: "회사 팀"이 아니라 **친구를 초대해 같이 집중**하는, 개인 중심의 소셜 집중 모드.

## 무엇이 바뀌나 (개선 요약)

| 항목 | 기존 플랜-팀모드 | 개선판 |
|---|---|---|
| 컨셉 | 팀(team/회사) 공유 모닥불 | **친구 초대** "함께 집중하기" (개인 중심) |
| 명칭 | 팀 / 팀 코드 / 팀 목표 | **모닥불 모임 / 초대 코드·링크 / 우리 목표** |
| 정체성 | 익명 `X-User-Id` + 클라 닉네임 `fc_name` | **로그인 계정(user_id + display_name)** 그대로 사용 |
| 멤버십 영속성 | 새로고침 시 클라가 기억(불안정) | **서버가 user_id 로 소속 기억** → 어느 기기서든 유지 |
| 이름 표시 | 클라가 보낸 이름(위조 가능) | **로그인된 표시 이름**(서버 신뢰) |
| 초대 | 6자 코드 | 6자 코드 **+ 공유 가능한 초대 링크** |
| 규모 | "팀" 뉘앙스(대규모) | **소규모 친구 모임**(스터디/메이트) 강조 |

## 왜 이게 개인 생산성에 더 맞나
- 혼자 하면 작심삼일 → **친구 한두 명과 같이** 하면 지속률↑ (소셜 책임감)
- "회사 팀 KPI"가 아니라 **내 집중을 친구와 공유·응원**하는 개인적 동기부여
- 로그인 덕분에 **내 친구 모임이 계정에 묶여** 폰·노트북 어디서든 이어짐

---

## 로그인 연계 설계 (핵심 변경)

1. **정체성 소스 교체**
   - 기존: `X-User-Id`(localStorage UUID) + 요청 바디의 `user_name`(클라 입력)
   - 개선: 로그인 시 받은 `user_id`(서버 발급, `u-xxxx`) 를 `X-User-Id` 로 전송(이미 구현됨).
     이름은 **서버가 user_id→display_name 으로 조회**해 채움. 클라가 보낸 이름은 신뢰하지 않음.
   - 효과: 리더보드/접속자 이름이 위조 불가, 테스터 모드도 `tester-demo` 계정으로 자연 동작.

2. **소속 영속성**
   - `team_store` 에 `user_id -> set(team_id)` 역인덱스 추가.
   - 새 엔드포인트 `GET /api/teams/mine` → 로그인 사용자가 속한 모임 목록.
   - 프론트: 앱 진입 시 `mine` 조회 → 모임 자동 복원(클라 localStorage 의존 제거).

3. **게스트/테스터 처리**
   - 테스터(`tester-demo`)도 모임 생성·가입 가능(데모용). 단 안내 문구로 "예시 계정" 표기.

## 작업 목록 (기존 10태스크 → 개선 매핑)

### Backend
- **T1. 모임 저장소** (`team_store.py`, `team_models.py`)
  - `Member{user_id, name, joined_at}` — name 은 **저장 시 auth_store 에서 조회**해 채움.
  - `Group{id, code, name, owner_id, created_at, members}` (기존 Team 명칭 → Group/모임)
  - `create(name, owner_id)`, `join(code, user_id)`, `get`, `find_by_code`
  - **추가:** `mine(user_id) -> list[Group]` (역인덱스)
  - name 조회를 위해 `auth_store` 에 `display_name(user_id)` 헬퍼 추가.
- **T2. 세션에 group_id 연계 + 모임 요약**
  - `SessionCreate` 에 `group_id: Optional[str]` 추가(하위호환).
  - `user_id` 는 바디가 아니라 **헤더 X-User-Id** 에서 취득(신뢰).
  - `summary(group_id)` → 총 집중분, 우리 목표 진행률, 모닥불 레벨, **리더보드(이름은 서버 조회)**, 👑 MVP.
- **T3. 실시간 접속(heartbeat + TTL 60초)** — 이름은 서버 조회.
- **T4. 모임 라우터** `/api/teams` (create/join/**mine**/summary/presence/feed) — `main.py` 등록.

### Frontend
- **T5. 정체성 헬퍼 정리**
  - `getUserName()`/`fc_name` 의존 제거 → `getDisplayName()`(로그인값) 사용.
  - api: `groups.create/join/mine/summary/presence/feed`, 세션 생성 시 `group_id` 동봉.
- **T6. 친구 초대/참여 플로우** (`InviteGate.jsx`)
  - "모임 만들기" / "코드로 참여" / **"초대 링크 복사"**(`?invite=CODE`).
  - 앱 로드시 URL 의 `?invite=` 자동 채움.
- **T7. 함께 타는 모닥불 대시보드** (`GroupDashboard.jsx`)
  - 공유 `FocusScene` + 우리 목표 게이지 + 리더보드(이름·집중분) + 👑 MVP.
- **T8. 실시간 접속/불씨 피드**(5초 폴링, 탭 숨김 시 중지) — 집중 중 heartbeat.
- **T9. 메뉴 연결** 사이드바에 `🏕️ 같이 집중` 추가 → `GroupView`.
- **T10. README + 검증**(pytest + build, 2계정 스모크).
