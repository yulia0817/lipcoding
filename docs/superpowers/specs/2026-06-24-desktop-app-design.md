# Focus Scene 데스크탑 앱 전환 — 설계 문서

- 작성일: 2026-06-24
- 상태: 설계 확정 (구현 계획 대기)
- 대상: 기존 웹앱 `lipcoding/`(FastAPI + React) → 맥 데스크탑 앱

## 1. 배경 / 목적

기존 Focus Scene은 음성 기반 포모도로 **웹앱**으로, 기능이 많고(팀·게임·깃헙·스탠드업 등)
브라우저 탭으로 띄워 쓰기엔 불편하다. 일상에서 늘 곁에 두고 쓰는 **가벼운 맥 데스크탑
집중 타이머**가 목표다. 레퍼런스는 "맥 메뉴바/노치에 떠서 시간을 재는" 류의 상주형 앱.

**핵심 가치:** 항상 보이는 집중 타이머 + 모닥불 분위기 + 내 집중 기록이 영구 보존.

## 2. 범위 (Scope)

### 유지 ✅
- 집중 타이머 (`usePomodoro`): 프리셋/직접입력, 세션 중 시간 변경
- 모닥불 분위기: `FocusScene` / `Campfire` (CSS 애니메이션 그대로), 진행도에 따라 불꽃 강도
- 카테고리·태그 입력 (텍스트)
- 개인 기록: 날짜별 기록 (`DailyView`)
- 통계: 연속 집중일(streak), 12주 히트맵, 집중 vs 딴짓 비율 (`StatsView`)
- 스킨 테마 7종 (모닥불/벽난로/해변/숲/비/우주/오로라)
- 휴식 스트레칭 코치 (`BreakCoach`) — 음성 출력(TTS, `speechSynthesis`)은 데스크탑 WebView에서 동작
- 배경 사운드(ambient)

### 제거 ❌
- 로그인/회원가입/테스터 (`auth`, `identity`, `LoginView`)
- 같이 집중 / 팀 / 리더보드 / 초대 (`team`, `useGroups`, Group/Invite 뷰)
- 게이미피케이션: 레벨/XP/장작(코인)/일일 퀘스트/배지 (`gamify`, `useProfile`의 게임 부분)
- GitHub 연동 (`github`, `GithubView`), AI 스탠드업 (`standup`, `StandupView`)
- 한 줄 회고(저널) / 일일 목표
- 음성 입력(받아쓰기, `useSpeech`의 SpeechRecognition) — WKWebView 미지원
- **백엔드 전체** (FastAPI/Python, `backend/`)

### 보류 / 선택 (MVP 이후)
- 전체화면 몰입 모드(`ImmersiveScene`): `FocusScene` 재사용이라 비용은 낮지만, MVP는
  메인 창 + 메뉴바에 집중. 코어가 안정된 뒤 선택적으로 다듬는다.

### 의도적 비목표 (Non-goals, 지금은 안 함)
- 클라우드 동기화 / 멀티 디바이스
- 크로스플랫폼(윈도/리눅스) — 우선 **맥 전용**
- 네이티브 음성 인식 (추후 별도 검토 가능)
- 외부 배포용 코드서명/공증 (개인 사용 우선)

## 3. 기술 스택 / 결정 사항

| 항목 | 결정 | 이유 |
|---|---|---|
| 셸 | **Tauri** (Rust) | 가벼운 번들/메모리, 시스템 WebView, 좋은 트레이 지원 |
| UI | 기존 **React + Vite** 재사용 | 모닥불·통계 UI 그대로 활용, JS 자산 보존 |
| 상주 표면 | **메뉴바 트레이** | "항상 보임"을 가장 단순하게 충족 |
| 저장 | **로컬 JSON 파일** | 오프라인, 영구 보존, 백엔드 불필요 |
| 음성 입력 | **제거(텍스트만)** | WKWebView가 SpeechRecognition 미지원 |
| 플랫폼 | **맥 전용** | 메뉴바 중심 UX |

## 4. 아키텍처

```
┌─ Tauri 셸 (Rust, 맥 메뉴바 앱) ─────────────────┐
│  • 트레이 아이콘/타이틀: 🔥 24:32 (집중 중 매초 갱신)  │
│  • 트레이 메뉴: 시작 / 일시정지 / 종료 · 열기 · 끝내기  │
│  • 메인 창 show/hide 토글                          │
│  • 앱 데이터 디렉터리 경로 제공 (저장 위치)            │
│                                                   │
│  ┌─ React + Vite (기존 UI 재사용, 코어만) ───────┐ │
│  │  메인 창: 모닥불 + 타이머 + 카테고리 입력         │ │
│  │           + 날짜별 기록 + 통계                  │ │
│  │  타이머 로직(JS) = 진실의 원천                   │ │
│  │   → 매 틱마다 남은시간을 Tauri 커맨드로 트레이에 push │ │
│  │   → 세션 완료 시 localStore 에 append           │ │
│  └────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────┘
            ↕  로컬 JSON 파일 (앱 데이터 폴더)
        sessions · settings   (← api.js 대체: localStore.js)
```

### 컴포넌트 경계
- **타이머는 프론트(JS)가 소유**한다. 매초 남은 시간(문자열)을 Tauri 커맨드
  `set_tray_title`로 트레이에 반영. 메인 창을 닫아도(hide) 트레이 덕에 프로세스/웹뷰가
  살아 있어 타이머는 계속 진행된다.
- **`localStore.js`** 가 기존 `api.js`(백엔드 호출 32개 함수)를 대체한다. 디스크 읽기/쓰기 +
  통계 계산을 담당. streak/히트맵/집중비율은 저장된 세션에서 **런타임 계산**(따로 저장 안 함).
- **Rust 측은 얇게** 유지: 트레이/메뉴/창 토글/저장경로만. 비즈니스 로직 없음.

### 모듈 단위 책임
| 모듈 | 한 일 | 의존 |
|---|---|---|
| `usePomodoro` (기존) | 카운트다운, 일시정지/재개, 시간 변경 | 없음 (순수) |
| `localStore.js` (신규) | 세션 저장/로드, 통계 파생 계산 | Tauri fs/path |
| `trayBridge.js` (신규) | 남은시간 → 트레이 타이틀 push, 트레이 메뉴 ↔ 타이머 명령 연결 | Tauri invoke/event |
| Rust `lib.rs` | 트레이 생성, 메뉴 핸들러, 창 show/hide | tauri |

## 5. 데이터 모델 (로컬 JSON)

저장 위치: Tauri `appDataDir()` 하위 `focus-scene/store.json`.

```jsonc
{
  "version": 1,
  "sessions": [
    {
      "id": "string",
      "task": "string",          // 비우면 카테고리 이름이 라벨로 들어감
      "category": "공부",
      "tags": ["코테"],
      "startedAt": "2026-06-24T09:00:00.000Z",
      "durationMin": 25,
      "completed": true,
      "focusSec": 1500,
      "distractedSec": 0
    }
  ],
  "settings": {
    "focusMin": 25,
    "breakMin": 5,
    "skin": "campfire",
    "ambient": "fire"
  }
}
```

- 파생값(오늘 집중분, streak, 12주 히트맵, 집중/딴짓 비율)은 저장하지 않고 `localStore`가
  세션 배열에서 계산한다. (기존 `session_store.py`의 집계 로직을 JS로 이식)
- 쓰기 시점: 세션 완료 시, 설정 변경 시. 디바운스 후 파일 저장.

## 6. 메뉴바(트레이) 동작

- **표시:** 집중 중 `🔥 24:32`, idle `🔥`.
- **좌클릭:** 메인 창 토글(show/hide).
- **메뉴:** `시작 / 일시정지·재개 / 종료` · `열기` · `앱 종료`.
- (선택, 추후) 트레이 팝오버에서 빠른 시작 — MVP 이후.

## 7. 마이그레이션 단계 (개략)

> 상세 구현 순서·체크포인트는 다음 단계(writing-plans)에서 작성.

1. **스캐폴드:** 기존 `frontend/` 위에 Tauri 셸 추가 → 앱이 모닥불 메인 화면으로 부팅.
2. **코어만 남기기:** 팀/깃헙/스탠드업/게임/로그인/저널/일일목표 파일·네비·라우트 제거.
   `useSpeech`의 받아쓰기 제거(텍스트 입력만). 백엔드 없이 구동되도록 `api.js` 분리.
3. **로컬 영속화:** `localStore.js` 작성, 세션/통계/날짜별 기록을 연결. `session_store.py`
   집계 로직(streak/heatmap/ratio/daily)을 JS로 이식하고 동등성 검증.
4. **메뉴바 트레이:** Rust 트레이 + `set_tray_title` 커맨드, `trayBridge.js`로 남은시간 push,
   메뉴 컨트롤 ↔ 타이머 연결, 창 토글.
5. **마무리:** 스킨/휴식코치(TTS)/설정 저장, 앱 아이콘, `.dmg`/`.app` 패키징(개인용).

## 8. 테스트 / 검증

- **유닛(Vitest):**
  - `usePomodoro` 카운트다운/시간변경 계산.
  - `localStore` 저장·로드 라운드트립, 통계 파생(streak/heatmap/ratio)이 기존 파이썬
    집계와 동일 결과인지 (대표 케이스 픽스처로 비교).
- **수동 검증:**
  - 세션 완료 → 앱 재시작 후 기록·streak 유지.
  - 집중 중 메인 창 닫아도 트레이 시간이 매초 갱신.
  - 트레이 메뉴의 시작/일시정지/종료가 타이머와 동기화.
  - 스킨 전환, 휴식 코치 TTS 동작.

## 9. 리스크 / 미해결

- **음성 받아쓰기 상실:** WKWebView 한계. 텍스트 입력으로 대체(합의됨).
- **Rust 툴체인 필요:** 최초 셋업 비용. (설치 안내 포함)
- **트레이 타이틀 갱신 비용:** 매초 invoke. 문자열만 갱신이라 부담 작으나, 필요 시
  Rust 측 타이머로 이동 가능(차선책).
- **패키징/서명:** 개인 사용은 미서명 `.app`로 충분. 외부 배포 시 별도.
